# Frontend Design Product

Use this skill when a rough product idea needs to become a usable, polished frontend experience.

## Product Design Lens

Before writing code, decide:

- Primary user action: import personality material, answer questions, generate a persona, create an image, save/share a result, or refine output.
- Trust moment: explain privacy, clarify that the result is creative/personality insight rather than clinical diagnosis, and show how inputs are used.
- Conversion moment: guide the user from curiosity to first generated result with minimal friction.

## Page Patterns

- Homepage: show the core product in the first viewport, not just abstract marketing copy.
- Input page: support paste text, upload notes, choose style, and optional guided questions.
- Analysis page: show trait summary, strengths, tensions, communication style, creative archetype, and image prompt ingredients.
- Character result page: show persona card, visual direction, prompt, editable sliders/tags, and regenerate controls.
- Dashboard: show saved personas, recent generations, filters, and quick compare.

## Implementation Rules

- Ship working files, not descriptions only.
- Use restrained sections and clear hierarchy. Avoid nested cards and clutter.
- Include responsive behavior for mobile and desktop.
- Add clear empty, loading, partial-result, and error states.
- Write user-facing copy in natural language. Avoid overclaiming psychological accuracy.

## Safety Positioning

Frame outputs as exploratory and creative. Do not present generated personality profiles as medical, diagnostic, or fixed identity judgments.

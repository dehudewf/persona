# SOUL.md Maker

Use this skill to transform messy personality input into a structured persona profile that can drive copy, UI results, and character image prompts.

## Inputs To Accept

- Freeform self-description
- Chat logs or writing samples
- Interview answers
- Fictional character notes
- MBTI/Big Five-like preferences, when provided by the user
- Desired visual style or role

## Persona Profile Structure

Create outputs with these sections:

- Core identity: short name, archetype, one-line essence
- Surface signals: tone, vocabulary, pacing, energy
- Inner drivers: needs, values, fears, motivations
- Social style: how the persona communicates and reacts
- Strengths: useful tendencies and advantages
- Tensions: internal conflicts or tradeoffs
- Decision pattern: how the persona chooses under pressure
- Visual direction: clothing style, environment, colors, props, symbols, and mood
- Image prompt ingredients: concise prompt-safe descriptors for generation
- Boundaries: what the profile should not claim or imply

## Interview Modes

Use Quick Forge when the user wants speed:

1. What is this persona's goal?
2. What are three obvious traits?
3. What kind of visual world should they belong to?
4. Should the result feel realistic, anime, cinematic, game-like, editorial, or abstract?

Use Deep Forge when the user wants detail:

1. Ask about values, conflicts, habits, speech, relationships, taste, and symbolic imagery.
2. Convert answers into the structured profile.
3. Separate observed input from creative inference.

## Safety Rules

- Do not diagnose mental health conditions.
- Do not claim the profile is a fixed truth about a real person.
- Do not judge physical attractiveness or rank bodies.
- For minors or youth-oriented personas, keep outputs non-sexual and age-appropriate.
- Treat personality output as creative interpretation and product content.

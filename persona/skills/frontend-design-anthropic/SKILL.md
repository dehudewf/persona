# Frontend Design Anthropic

Use this skill to create memorable frontend interfaces with a clear aesthetic direction and production-grade implementation.

## Core Workflow

1. Identify the screen's job: attract, explain, collect input, show analysis, or generate a character image.
2. Choose one strong visual direction before coding. Good directions for this project include editorial psychology report, cinematic character lab, soft diary-like profile, futuristic personality scanner, or refined creative studio.
3. Define the memorable element: for example a personality radar, character card, generated portrait stage, timeline of traits, or dramatic result reveal.
4. Implement working UI with semantic HTML, accessible controls, responsive layout, and realistic states.
5. Verify the first viewport immediately communicates the product: personality input, analysis, and generated persona/image output.

## Design Rules

- Avoid generic gradient hero sections, identical three-card grids, and vague placeholder copy.
- Make typography, spacing, color, and motion choices intentional.
- Use real product states: empty, loading, analyzing, generated, error, and regeneration.
- Keep the interface safe and non-clinical: present personality output as creative insight, not diagnosis.
- When showing a character image, avoid body-rating language and appearance judgment. Describe style, mood, setting, and symbolic details instead.

## Output Expectations

When creating code, deliver working files that can open in a browser or fit the existing app stack. Prefer concise, maintainable components and scoped CSS. Include enough sample content to make the personality-analysis flow feel real.
